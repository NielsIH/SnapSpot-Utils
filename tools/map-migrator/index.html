    How to Use Map Migrator
    
      
    
        
    Step 1: Load Files
    
        
      
          
    • Drop your SnapSpot export (.json) on the left panel
      • 
          
    • Drop your target on the right panel:
            
        
              
      • Image file: Creates new export with transformed markers (Replace Mode)
        • 
              
      • SnapSpot export (.json): Merges transformed markers with existing markers (Merge Mode)
        • 
            
      
          
      • 
        
    

        
    Step 2: Select Reference Points
    
        
      
          
    • Click on a recognizable feature in the source map
      • 
          
    • Click the same feature in the target map
      • 
          
    • Repeat for at least 3 matching locations
      • 
          
    • Use landmarks like building corners, road intersections, etc.
      • 
          
    • Tip: Use mouse wheel to zoom in for precise placement
      • 
          
    • Tip: Hold Ctrl and drag to pan around the map
      • 
        
    

        
    Step 3: Calculate & Export
    
        
      
          
    • Click "Calculate Transformation" to compute the mapping
      • 
          
    • Review the metrics and warnings
      • 
          
    • Click "Preview" to see where markers will be placed
      • 
          
    • Click "Generate Export" to download the migrated file
      • 
          
    • Merge Mode: Preview shows which markers will be added vs merged (5px tolerance)
      • 
        
    

        
    Two Migration Modes
    
        
    Replace Mode (Target = Image)
    
        
      
          
    • Creates new export with transformed markers on new map image
      • 
          
    • All source markers transformed to target coordinates
      • 
          
    • Original photos linked to transformed markers
      • 
        
    

        
    Merge Mode (Target = Export)
    
        
      
          
    • Transforms source markers and merges with target markers
      • 
          
    • Choose duplicate detection strategy (default: none):
      • 
          
        
            
      • None: Add all as new markers (safest, default)
        • 
            
      • Smart: Cascading detection (photos ‚Üí labels ‚Üí coordinates)
        • 
            
      • Photo Filenames: Match by 70%+ shared photo filenames
        • 
            
      • Label/Description: Match by text (case-insensitive)
        • 
            
      • Coordinates: Match within tolerance (based on transformation RMSE)
        • 
          
      
          
    • When duplicates detected: photos are merged, existing marker preserved
      • 
          
    • When no match: marker added as new to target export
      • 
        
    

        
    Why is 'none' the default? Transformed coordinates rarely match exactly. If you need duplicate detection, choose a strategy based on your markers (use Smart if markers have photos, Label if they have descriptive text).
